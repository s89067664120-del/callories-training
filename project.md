# CalBot — Телеграм-бот для подсчёта калорий

## Что это

CalBot — Telegram-бот, который помогает худеть без сложных таблиц и ручного подсчёта калорий. Пользователь фотографирует еду — бот сам распознаёт блюдо, считает КБЖУ и записывает в дневник. Главное отличие от приложений-конкурентов: всё происходит прямо в Telegram, бесплатно и за 30 секунд.

---

## Для кого

Люди, которые хотят похудеть, но не готовы вести подробный дневник питания вручную. Им нужен минимум усилий и максимум результата — без таблиц, без взвешивания каждого грамма, без сложных приложений.

---

## Проблема

Большинство людей, которые хотят похудеть, бросают попытки вести дневник питания в первые две недели — не потому что не хотят, а потому что это слишком трудоёмко: нужно вручную искать каждый продукт, взвешивать порции, разбираться в интерфейсе приложения. Специализированные приложения (MyFitnessPal, Lose It!) прячут функцию распознавания по фото за платный план от $40/год, а русскую кухню распознают плохо. В Telegram нет ни одного бота, который решает эту задачу качественно.

---

## Главная идея

Пользователь просто фотографирует еду — бот сам распознаёт блюдо, считает калории и записывает в дневник. Никаких ручных вводов, никаких меню из 10 шагов. Только кнопки.

> Минимум действий → максимум результата

---

## Конкурентное преимущество

Исследование 5 ключевых конкурентов (MyFitnessPal, Lose It!, Calorie Mama AI, Noom, SnapCalorie) выявило три незакрытые ниши:

1. **Нет Telegram-бота с фото-распознаванием** — ни один из конкурентов не предлагает этого формата
2. **Русская кухня не покрыта** — все западные продукты слабо распознают борщ, гречку, пельмени, оливье
3. **Фото-функция за платным барьером** — у лидеров (MFP, Lose It!) распознавание по фото доступно только в Premium от $40/год; CalBot даёт это бесплатно

## Функции (v1 — MVP)

### 1. Онбординг

Бот знакомится с пользователем и задаёт вопросы по одному:

- Рост
- Вес
- Возраст
- Цель: похудеть / поддержание / набор массы
- Уровень активности

На основе данных рассчитывает суточную норму калорий (TDEE) по формуле Миффлина–Сан Жеора и целевую норму для похудения.

После настройки сразу предлагает два действия: добавить еду или посмотреть прогресс.

> UX-требования: прогресс-бар "шаг 2 из 5", возможность пропустить шаг, результат (норма калорий) показывается сразу — до любого меню.

---

### 2. Распознавание еды по фото (ядро продукта)

Пользователь отправляет фото — бот отвечает:

- Что он видит на фото (список блюд)
- Примерную нутритивную ценность: калории, белки, жиры, углеводы
- Короткий совет по блюду (например, как немного снизить калорийность)
- Кнопки: **Добавить** / **Изменить** / **Не добавлять**

Пользователь не пишет текст — только нажимает кнопки.

> UX-требование: немедленный ответ "Анализирую..." после получения фото — без индикатора загрузки пользователи шлют фото повторно. Целевое время сценария: под 30 секунд.

> Сценарий ошибки: если блюдо не распознано — бот предлагает кнопки ручного ввода, не просит написать текст.

---

### 3. Дневник питания

После добавления блюда бот показывает текущий итог дня:

- Сколько калорий съедено
- Сколько осталось до нормы
- Короткая мотивационная реплика

---

### 4. Итог дня

Вечером бот сам пишет краткий отчёт:

- Итоговые калории за день
- Оценка дня (в норме / небольшой перебор / хороший результат)
- Предложение получить рекомендации на завтра

> UX-требование: учёт часового пояса пользователя. Пуш в неподходящее время — один из главных триггеров отписки.

---

### 5. AI-коуч (эмоциональная поддержка)

Бот периодически пишет сам — без запроса пользователя:

- Отмечает серии успешных дней
- Поддерживает после срывов без осуждения
- Создаёт ощущение живого общения, а не холодного трекера

> Тональность бота — ключевой дифференциатор от конкурентов. Noom удерживает пользователей именно за счёт психологического подхода, а не точности данных.

## Чего НЕ будет

- Ручного ввода веса каждого ингредиента
- Сложных меню и вложенных разделов
- Обязательной регистрации и email
- Точного подсчёта до грамма — только реалистичные оценки
- Строгих запретов и «плохих продуктов» — бот не осуждает

## Технический стек

| Компонент | Решение | Обоснование |
|---|---|---|
| Язык | Python 3.11+ | |
| Фреймворк бота | python-telegram-bot v21 | ConversationHandler ложится на пошаговый онбординг |
| Vision AI | Claude Vision (claude-sonnet) | Закрывает и распознавание еды, и AI-коуч одним API-ключом |
| AI-коуч | Claude API (тот же ключ) | |
| База данных | SQLite → PostgreSQL (Neon) | SQLite на старте, миграция при росте занимает 1–2 часа |
| ORM | SQLAlchemy | |
| Планировщик | APScheduler + SQLAlchemyJobStore | Задачи переживают перезапуски бота |
| Хостинг | Railway → Hetzner при росте | Railway: быстрый старт + встроенная БД; Hetzner: дешевле при 200+ пользователях |

**Бюджет на старте:** ~$10/мес (Railway $5 + Claude API ~$5 при ~1000 запросов)

**При 100 активных пользователях × 3 фото/день:** ~$50/мес, основная статья расходов — Claude API

## Монетизация

### Стратегия по этапам

**Фаза 1 (0 → 500 пользователей):**
Lifetime Deal $19–25 (Telegram Stars) + 1 affiliate-партнёрка спортпита
→ Цель: $500–2,000 разово + валидация спроса

**Фаза 2 (500 → 2000 пользователей):**
Подписка $4.99/мес для новых + affiliate + первый марафон похудения
→ Цель: $400–1,200 MRR + $500–1,500 от марафона

**Фаза 3 (2000 → 5000 пользователей):**
Подписка + upsell AI-планов питания ($7.99) + affiliate + нативная реклама
→ Цель: $1,300–3,900 MRR

### Freemium-деление

| Бесплатно | Premium ($4.99/мес) |
|---|---|
| До 5 записей еды в день | Неограниченное логирование |
| Базовые КБЖУ | Детальные микронутриенты |
| Ручной ввод | Распознавание фото (AI) |
| Недельная статистика | Полная история + экспорт |
| — | Персональные рекомендации AI |
| — | Планировщик питания |

### Ключевые правила монетизации

- Триал 3–7 дней конвертирует в 2–3x лучше, чем скидка 50%
- Не ставить paywall раньше чем через 2–3 дня использования
- Годовая подписка ($39.99) снижает churn в 4–5x по сравнению с месячной
- AI-функции (фото + персональный план) — лучший повод платить

## UX-принципы

1. **Правило 3 тапов** — добавить еду не более чем за 3 действия
2. **Немедленный feedback** — ответ на любое действие в течение 2 секунд
3. **Кнопки везде** — пользователь не должен писать текст в повседневных сценариях
4. **Поддержка, не аудит** — после срыва бот поддерживает, а не осуждает
5. **Учёт часового пояса** — вечерний отчёт приходит вечером, а не в 3 ночи
6. **Прогрессивное раскрытие** — сложные функции появляются по мере привыкания
7. **Прощение ошибок** — отмена последней записи доступна всегда

## Метрики успеха MVP

- Пользователь добавляет еду хотя бы 3 дня подряд
- Процент открытых ответов бота > 60%
- Пользователи возвращаются на следующий день после первого входа
- Retention на 7-й день > 30%
- Retention на 30-й день > 15%

## Первые шаги

1. **Настройка окружения** — создать бота через @BotFather, получить токен; завести аккаунт Anthropic и получить API-ключ Claude
2. **Скелет бота** — поднять проект на python-telegram-bot v21, настроить ConversationHandler для онбординга
3. **Онбординг** — реализовать сбор данных (рост, вес, возраст, цель, активность) и расчёт TDEE
4. **Фото-распознавание** — подключить Claude Vision: принять фото → получить КБЖУ → показать пользователю с кнопками
5. **Дневник** — сохранять записи в SQLite, показывать счётчик дня после каждого добавления
6. **Вечерний отчёт** — настроить APScheduler, отправлять итог дня с учётом часового пояса
7. **AI-коуч** — добавить триггеры для мотивационных сообщений (серии, срывы)
8. **Деплой** — выгрузить на Railway, проверить работу в реальных условиях
9. **Тест с 5–10 реальными пользователями** — собрать обратную связь до добавления новых функций
