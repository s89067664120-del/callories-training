# CLAUDE.md — Правила работы с проектом CalBot

## Что это за проект

CalBot — Telegram-бот для подсчёта калорий по фото еды. Пользователь фотографирует блюдо, бот распознаёт его через Claude Vision и записывает КБЖУ в дневник питания.

## Структура проекта

```
callories-training/
├── calbot/                  ← весь код бота
│   ├── bot/
│   │   ├── main.py          ← точка входа, запуск бота
│   │   ├── handlers/
│   │   │   ├── onboarding.py  ← онбординг (5 вопросов + TDEE)
│   │   │   ├── food.py        ← фото → Claude Vision → КБЖУ
│   │   │   └── diary.py       ← дневник питания
│   │   ├── services/
│   │   │   ├── claude.py      ← Claude Vision API + AI-коуч
│   │   │   └── calculator.py  ← расчёт TDEE (Миффлин–Сан Жеор)
│   │   ├── database/
│   │   │   ├── models.py      ← модели User, FoodEntry
│   │   │   └── db.py          ← SQLite подключение

│   │       └── jobs.py        ← вечерний отчёт в 21:00 МСК
│   ├── .env                 ← токены (не коммитить!)

```

## Окружение

- **Python:** 3.14 (локально), 3.11 (на Render)
- **Запуск локально:** `/c/Users/р/AppData/Local/Python/bin/python.exe -m bot.main` из папки `calbot/`
- **База данных:** SQLite локально (`calbot.db`), создаётся автоматически при первом запуске
- **Хостинг:** Render (бесплатный Web Service), мониторинг — UptimeRobot каждые 5 минут

## Переменные окружения

Хранятся в `calbot/.env` (не коммитится в Git).

```
TELEGRAM_BOT_TOKEN=...   # от @BotFather в Telegram
ANTHROPIC_API_KEY=...    # от console.anthropic.com
PORT=10000               # по умолчанию, Render подставляет автоматически
```

## Как деплоить изменения

1. Внести изменения в код
2. Проверить синтаксис: `python -m py_compile bot/**/*.py`
3. Закоммитить и запушить:
   ```
   git add -A
   git commit -m "описание изменений"
   git push
   ```
4. Render автоматически подхватит изменения и перезапустит бота

## Важные технические решения

- **Веб-сервер aiohttp** запускается параллельно с ботом — нужен для Render Free tier (иначе сервис усыпляется)
- **ConversationHandler** управляет онбордингом — состояния: `WELCOME → HEIGHT → WEIGHT → AGE → GOAL → ACTIVITY`
- **APScheduler + SQLAlchemyJobStore** — задачи вечернего отчёта переживают перезапуски
- **Claude Vision** используется для двух задач: распознавание еды и генерация мотивационных сообщений (один API-ключ)
- **asyncio.run_in_executor** — Claude Vision вызывается в отдельном потоке чтобы не блокировать бота

## Что нельзя делать

- Коммитить `.env` — в нём реальные токены
- Коммитить `calbot.db` — база данных с данными пользователей
- Использовать синхронные вызовы Claude внутри async-обработчиков напрямую — только через `run_in_executor`
- Запускать двух экземпляров бота одновременно — Telegram выдаст ошибку Conflict

## Модели Claude

- Распознавание фото и AI-коуч: `claude-opus-4-5`
- При необходимости снизить стоимость можно переключить на `claude-haiku-4-5-20251001`
