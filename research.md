**Ссылка:** https://www.myfitnesspal.com

**Что умеет:**
- Фото-распознавание блюд (только Premium)
- База данных 14+ млн продуктов

## 3. UX и сценарии

### Сценарий 1: Первое знакомство (онбординг)

**Шаги пользователя:**
1. Нажимает /start → видит приветствие с обещанием ("за 1 минуту")
2. Бот задаёт вопросы по одному — рост, вес, возраст, цель, активность
3. Получает свою норму калорий и сразу видит кнопки действий

**Что удобно:** прогресс-бар "шаг 2 из 5", возможность пропустить шаг, CTA сразу после показа TDEE

**Что бесит:** длинный wizard до первого полезного действия, нет прогресс-индикатора, нельзя вернуться назад

**Типичные ошибки разработчиков:**
- Задавать все вопросы одним сообщением
- Не объяснять, зачем нужны данные
- Не показывать результат сразу после онбординга (уводить в меню)
- Забыть о повторном онбординге при возвращении пользователя

---

### Сценарий 2: Быстрое добавление еды (ежедневный сценарий)

**Шаги пользователя:**
1. Открывает бот → фотографирует тарелку
2. Отправляет фото
3. Видит анализ (блюдо, КБЖУ, совет) и нажимает "Добавить"
4. Получает обновлённый счётчик дня

**Целевое время сценария: под 30 секунд**

**Что удобно:** немедленное "Анализирую..." сразу после фото, кнопки вместо текстового ввода, виден остаток калорий после добавления

**Что бесит:** задержка без индикатора загрузки, неверное распознавание без возможности быстро исправить, "протухшие" кнопки если долго не нажимал

**Типичные ошибки разработчиков:**
- Не обрабатывать нефудовые фото (скриншоты, мемы) — бот зависает
- Не давать возможность исправить порцию одним кликом
- Не сохранять контекст для кнопок — кнопки работают только в момент получения

---

### Сценарий 3: Просмотр прогресса / мотивация

**Шаги пользователя:**
1. Вечером получает пуш с итогом дня
2. Видит цифры + мотивационный комментарий
3. Опционально — рекомендации на завтра

**Что удобно:** пассивный сценарий (пуш) эффективнее активного (кнопка "прогресс"), поддержка после срывов вместо осуждения, текстовый прогресс-бар лучше PNG-графиков

**Что бесит:** пуш в неподходящее время (нужен учёт часового пояса!), только цифры без эмоций, одинаковые сообщения каждый день

**Типичные ошибки разработчиков:**
- Пуши без учёта часового пояса пользователя
- После срыва — осуждающий тон вместо поддержки
- Нет вариативности сообщений — пользователь перестаёт читать

---

### 7 UX-принципов для CalBot

1. **Правило 10 секунд** — ежедневное действие (добавить еду) не более 3 тапов
2. **Экономия сообщений** — убирать пустые подтверждения ("Хорошо!"), оставлять обновления состояния ("Сегодня: 900/1500 ккал")
3. **Немедленный feedback** — любое действие пользователя получает ответ в течение 2 секунд
4. **Прощение ошибок** — всегда есть возможность отменить последнюю запись, изменить профиль
5. **Поддержка, не аудит** — тональность бота напрямую влияет на retention
6. **Прогрессивное раскрытие** — сначала простые функции, сложные — по мере привыкания
7. **Пуши как инструмент возврата** — максимум 1 в день, только с ценностью, с учётом часового пояса

### Зоны риска в текущей спецификации

1. **Сценарий ошибки распознавания** — что делать, если бот не распознал блюдо? Нужны кнопки, не свободный текст
2. **Часовой пояс** — вечерний отчёт должен учитывать регион пользователя
3. **Повторный онбординг** — что видит пользователь, который не заходил 30 дней?

## 4. Технологии

### 4.1 Распознавание еды (Vision AI)

| Сервис | Плюсы | Минусы | Цена |
|---|---|---|---|
| **Claude Vision** (Anthropic) | Отличный анализ контекста, хорошо понимает сложные блюда, тот же API для AI-коуча | Оценка нутриентов приблизительная (нет базы данных) | ~$3–15/млн токенов |
| **GPT-4o Vision** (OpenAI) | Сравнимое качество, широкая экосистема | Дороже при объёме, тот же минус с точностью нутриентов | ~$5–15/млн токенов |
| **Logmeal API** | Специализированная модель, возвращает структурированные данные с граммами | Ориентирована на западную кухню, слабо для СНГ | от $49/мес |
| **Calorie Mama API** | Хорошо для азиатской кухни, есть готовый API | Слабо для русской кухни, нишевый продукт | от $29/мес |

**Рекомендация:** Claude Vision — основной Vision API. Закрывает и распознавание, и AI-коуч одним ключом. При необходимости точных нутриентов — дополнить Nutritionix API.

---

### 4.2 База данных

| БД | Плюсы | Минусы | Цена |
|---|---|---|---|
| **SQLite** | Нулевая настройка, достаточна для сотен пользователей | Не подходит для >1000 одновременных пользователей | Бесплатно |
| **PostgreSQL** (Neon/Supabase) | Надёжность, масштабируемость, бесплатный тир | Требует настройки | Бесплатно до 0.5GB |
| **MongoDB** | Гибкая схема | Избыточен для структурированных данных питания | от $0 (Atlas) |

**Рекомендация:** Начать с SQLite + SQLAlchemy ORM. Миграция на PostgreSQL (Neon) при росте займёт 1–2 часа.

---

### 4.3 Хостинг

| Платформа | Плюсы | Минусы | Цена |
|---|---|---|---|
| **Railway** | Быстрый старт, встроенная PostgreSQL, автодеплой из GitHub | Дороже VPS при росте | от $5/мес |
| **Render** | Бесплатный тир | Усыпляет сервис на бесплатном плане — polling-бот перестаёт работать | от $0/бесплатно, $7/мес paid |
| **Hetzner CX22** | Лучшая цена/мощность, полный контроль | Требует настройки сервера | €4.35/мес |

**Рекомендация:** Railway для старта → Hetzner при 200+ активных пользователях.

---

### 4.4 Фреймворк для бота

| Фреймворк | Плюсы | Минусы |
|---|---|---|
| **python-telegram-bot v21** | Лучшая документация, ConversationHandler идеален для онбординга | Синхронный стиль в некоторых частях |
| **aiogram 3** | Async из коробки, популярен в рунете | Меньше туториалов на английском |
| **telebot (pyTelegramBotAPI)** | Простейший старт | Меньше возможностей для сложных сценариев |

**Рекомендация:** python-telegram-bot v21 — `ConversationHandler` точно ложится на пошаговый онбординг.

---

### 4.5 Планировщик задач

| Инструмент | Плюсы | Минусы |
|---|---|---|
| **APScheduler** | Работает в процессе бота, простая настройка | Задачи теряются при перезапуске без persistency |
| **APScheduler + SQLAlchemyJobStore** | Задачи переживают перезапуски | Чуть сложнее настройка |
| **Celery + Redis** | Мощное решение | Избыточно для MVP |

**Рекомендация:** APScheduler + SQLAlchemyJobStore — задачи хранятся в той же БД и переживают перезапуски.

---

### 4.6 Рекомендуемый MVP-стек

| Компонент | Решение |
|---|---|
| Язык | Python 3.11+ |
| Фреймворк бота | python-telegram-bot v21 |
| Vision AI | Claude Vision (claude-sonnet) |
| AI-коуч | Claude API (тот же ключ) |
| База данных | SQLite → PostgreSQL (Neon) |

## 5. Монетизация

### 5.1 Freemium + подписка

**Типичное деление:**

| Бесплатно | Платно |
|---|---|
| Подсчёт калорий (до 5 продуктов/день) | Неограниченное логирование |
| Базовые КБЖУ | Детальные микронутриенты |
| Ручной ввод | Распознавание фото (AI) |
| Недельная статистика | Полная история + экспорт |
| — | Персональные рекомендации AI |
| — | Рецепты и планировщик питания |

**Рекомендуемые цены:** $4.99/мес или $39.99/год

**Доход на 1000 активных пользователей:**
- Конверсия 3%: ~$150/мес
- Конверсия 5%: ~$250/мес
- Конверсия 8% (хороший онбординг): ~$400/мес

---

### 5.2 Разовая покупка vs подписка

| Модель | Плюсы | Минусы | Конверсия |
|---|---|---|---|
| Lifetime Deal ($19–29) | Высокая конверсия, быстрый кэш | Нет recurring revenue | 8–15% |
| Месячная подписка ($4.99) | Предсказуемый MRR | Ниже конверсия | 2–5% |
| Годовая подписка ($39.99) | Снижает churn в 4–5x | Выше барьер | 3–7% |

**Рекомендуемая стратегия:**
- 0–6 месяцев: LTD за $19–25 → быстрая валидация и первые деньги
- После 1000+ платных: переход на подписку для новых пользователей

---

### 5.3 Партнёрства

| Направление | Механика | Доход на 1000 пользователей |
|---|---|---|
| **Спортпит** (iHerb, Myprotein, 4fresh) | Affiliate 5–15% от заказа | $80–200/мес при активном продвижении |
| **Доставка здоровой еды** (Grow Food, Level Kitchen) | CPA $3–15 за первый заказ | ~$160/мес при 2% конверсии |
| **Фитнес-клубы** | Реферальная программа + white-label | $200–500/мес за белый лейбл |
| **Нутрициологи (B2B)** | $30–100/мес за профессиональный аккаунт | Отдельный B2B-продукт |

---

### 5.4 Дополнительные услуги

| Услуга | Цена | Конверсия | Доход на 1000 пользователей |
|---|---|---|---|
| AI-план питания (авто) | $4.99–9.99 разово | 7% | ~$560/мес |
| Консультация нутрициолога | $25–50 за сессию | 2% | ~$70/мес |
| Марафон похудения (30 дней) | $15–29 за участие | — | $500–2000 за запуск |
| Premium-гайды и рецепты | $4.99–24.99 | 5% | ~$150/мес |

---

### 5.5 Пошаговая стратегия монетизации

**Фаза 1 (0 → 500 пользователей):**
Lifetime Deal $15–25 + 1 affiliate-партнёрка спортпита
→ Ожидаемый доход: $500–2,000 разово

**Фаза 2 (500 → 2000 пользователей):**
Месячная подписка $4.99 + 1–2 affiliate + первый марафон
→ Ожидаемый MRR: $400–1,200 + $500–1,500 от марафона

**Фаза 3 (2000 → 5000 пользователей):**
Подписка + upsell AI-планы + affiliate + нативная реклама
→ Ожидаемый MRR: $1,300–3,900/мес

---

### 5.6 Ключевые инсайты

- Триал (3–7 дней) конвертирует в 2–3x лучше, чем скидка 50%
- AI-функции — лучший повод платить (фото + персональный план)
- Годовая подписка снижает churn в 4–5x
- Не ставить paywall до того, как пользователь получил ценность (первые 2–3 дня)
- Марафоны — лучший growth hack: вирусность + конверсия + retention одновременно
- Цены $9.99/мес для Telegram-бота — слишком высоко, оптимум $4.99/мес

### 6.1 FatSecret Bot (@FatSecret_bot)

**Что делает:** Telegram-интерфейс к крупнейшей базе данных продуктов FatSecret (9 млн+ позиций). Пользователь вводит название блюда текстом — бот возвращает КБЖУ. Логирует приёмы пищи по категориям (завтрак/обед/ужин/перекус).

**Сильные стороны:**
- Огромная база: 9 млн+ продуктов, включая брендированные товары
- Полная разбивка КБЖУ + клетчатка
- Многоязычный поиск, частично работает по-русски
- Полностью бесплатный
- Не нужно устанавливать отдельное приложение

**Слабые стороны:**
- Исключительно текстовый UX — никакого визуала, графиков, диаграмм
- Нет фото-распознавания
- Нет сканирования штрихкодов
- Нет постановки целей и трекинга веса
- Бот "забывает" контекст между сессиями
- Нет итогов за неделю/месяц с графиками

**Что мы делаем лучше:**
- Фото вместо текстового ввода — 0 усилий для пользователя
- Полноценный Mini App с визуальным дневником
- Трекинг веса + графики прогресса
- Голосовой ввод

### 6.2 Яедок (@yaedok_bot)

- Калькулятор нормы (рост/вес/возраст/цель/активность)
- Счётчик серий для мотивации
- Русскоязычный UX, неформальный тон
- Интегрированы каналы с советами и рецептами

**Слабые стороны:**
- Только русский язык
- Нет фото-распознавания
- Нет сканирования штрихкодов
- Mini App минималистичен по сравнению с полноценными мобильными приложениями
- Нет интеграции с фитнес-трекерами
- Расширенная аналитика — только в платной версии

**Что мы делаем лучше:**
- Фото-распознавание + голосовой ввод
- Учёт тренировок и дефицит/профицит в реальном времени
- Интеграция с шагомером телефона
- Рецепты и меню от AI

### 6.3 NutritionBot (@NutritionBot)

**Что делает:** Англоязычный бот для трекинга питания. Считает BMR, TDEE, предлагает разбивку макросов по целям (похудение/масса/поддержание). Сохраняет кастомные блюда. Присылает ежедневный отчёт по расписанию.

**Сильные стороны:**
- Встроенный калькулятор TDEE и BMR
- Рекомендации по макросам под разные цели (40/30/30 и т.д.)

- Ежедневный итог в заданное время
- Понятная навигация через inline-кнопки

**Слабые стороны:**
- Нет Mini App — только текст и кнопки, никакого визуала
- Меньшая база продуктов, чем FatSecret
- Нет фото-распознавания и штрихкодов
- Нет трекинга веса
- Онбординг очень длинный — много шагов до первого полезного действия
- Нет социальных функций

**Что мы делаем лучше:**
- Полноценный Mini App с визуальным прогрессом
- Фото-ввод как основной способ логирования
- Тренировки + баланс калорий (дефицит/профицит)

**Что мы делаем лучше:**
- Полноценный дневник питания за день/неделю/месяц

## 7. Экраны и элементы интерфейса

### 7.1 Онбординг (7–12 шагов)

| Шаг | Цель | Ключевые элементы |
|---|---|---|
| Приветствие / ценностное предложение | Зацепить одной фразой | Иллюстрация, слоган, кнопка «Начать», социальное доказательство |
| Выбор цели | Похудеть / поддержать / набрать | Крупные тап-карточки с иконками, анимация выбора |
| Текущий вес | Базовое измерение | Барабанный пикер, переключатель кг/фунты |
| Целевой вес | Конечная точка | Показать разницу, мотивирующий фрейм |
| Рост | Для расчёта BMR | Переключатель см/футы |
| Дата рождения / возраст | Для расчёта BMR | Простой ввод |
| Пол | Для расчёта BMR | Инклюзивный текст («для расчёта калорий») |
| Уровень активности | Коэффициент TDEE | 4–5 иллюстрированных вариантов |
| Темп похудения | 0.25 / 0.5 / 1 кг в неделю | Показать дефицит в ккал/день для каждого варианта |
| Итог: персональная норма | Дать ценность в конце онбординга | «Ваша цель: 1680 ккал/день» — это дофаминовый хит |
| Регистрация (опционально) | Сохранить данные | Telegram ID используется автоматически — упрощает этот шаг |

**Правило:** каждый вопрос повышает вовлечённость (эффект IKEA). Итоговый экран с нормой — лучшее место для пэйволла.

---

### 7.2 Основной экран — Дневник (P0)

Самый важный экран. Открывается несколько раз в день.

**Обязательные элементы:**
- **Полоса дат** вверху — свайп влево/вправо для навигации по дням
- **Кольцо калорий** — большая кольцевая диаграмма (потрачено vs норма), в центре — остаток
- **Строка макросов** — белки / жиры / углеводы в граммах и %, три мини-кольца или полоски
- **Секции приёмов пищи** — карточки «Завтрак / Обед / Ужин / Перекус»
  - В каждой карточке: список блюд с ккал, суммарные ккал, кнопка «+ Добавить»
- **Блок тренировок** — сожжённые ккал, чистый баланс дня (дефицит или профицит)
- **Счётчик воды** — стаканчики для быстрого добавления 250 мл
- **Счётчик серий** — «🔥 14 дней» в углу
- **Блок шагов** — если включена интеграция, показывает шаги и ккал сожжённых

---

### 7.3 Экран добавления еды (P0)

**Элементы:**
- Строка поиска (автофокус + иконки микрофона и штрихкода)
- Недавние блюда (до ввода текста)
- Частые блюда (логированные 3+ раз)
- Кнопка «Сфотографировать блюдо» — 1 тап, сразу в камеру
- Голосовой ввод — «Скажи, что ты ел»
- Результаты поиска: название, бренд, ккал/100г, размер порции
- Карточка блюда: выбор размера порции, полная таблица КБЖУ, кнопка «Добавить»

---

### 7.4 Экран фото-распознавания (P1)

**Флоу:**
1. Открывается камера с рамкой-оверлеем «Наведи на блюдо»
2. AI обрабатывает 1–3 сек (анимация сканирования)
3. Результаты: чипы с распознанными продуктами + ккал на фото
4. Под фото — карточки каждого элемента с редактируемой порцией
5. Кнопка «Добавить всё в [Приём пищи]»

**Правила UX:**
- Всегда маркировать как «приблизительно»
- Показывать диапазон, не одну цифру (~320–380 ккал)
- Давать fallback: «Не распознали? Найди вручную»
- Никогда не ждать больше 3 секунд без индикатора загрузки

---

### 7.5 Экран прогресса (P1)

- **График веса** — линейный, переключатели 1Н / 1М / 3М / Всё
- **Прогресс к цели** — «−3.2 кг из −8 кг» с прогресс-баром и датой достижения
- **Счётчик серий** — текущая и лучшая серия
- **Средние калории** — скользящее среднее за 7 дней vs норма
- **Тренд макросов** — недельное среднее белков/жиров/углеводов

---

### 7.6 Экран тренировок (P1)

- Поиск типа тренировки (бег, велосипед, йога, силовая и т.д.)
- Ввод длительности → автоматический расчёт сожжённых ккал
- История тренировок за неделю
- Интеграция с шагомером: шаги → ккал пассивного расхода
- Влияние на дефицит/профицит в реальном времени

### 7.8 Обязательные UI-компоненты

#### Кольцо калорий — самый важный элемент
- Большая кольцевая диаграмма: потрачено vs норма
- Центр: остаток калорий (или «превышение на X»)
- Цветовая схема: зелёный → жёлтый → красный
- Анимация заполнения при каждом добавлении
- Тап раскрывает разбивку по макросам

#### Полоски / кольца макросов
- Три полоски (Белки / Углеводы / Жиры) с граммами и %
- Стандартные цвета: синий = белки, оранжевый = углеводы, розовый = жиры
- Отображаются и в дневнике, и в карточке блюда

#### Карточки приёмов пищи
- Аккордеон с суммарными ккал заголовка
- Строки блюд: название, порция, ккал, свайп для удаления
- Кнопка «+ Добавить» всегда видна внизу карточки

#### Счётчик серий
- 🔥 + количество дней, всегда на виду
- Конфетти при продлении серии, кнопка поделиться
- «Ты на серии X дней!» — уведомление в Telegram

#### Экран пустого состояния
- Иллюстрация тарелки с дружелюбным текстом («Добавь первое блюдо!»)
- Никогда — обвинительный или пустой интерфейс

## 8. Ключевые механики удержания

### Геймификация, которая работает

| Механика | Реализация | Почему работает |
|---|---|---|
| Серия логирования | Дни подряд хотя бы с 1 блюдом | Эффект Duolingo — не хочется прерывать серию |
| Выполнение цели дня | Конфетти / анимация в конце дня | Переменное вознаграждение |
| Бейджи | «Первая запись», «7 дней», «−5 кг» | Разделяемые достижения |
| Еженедельный отчёт | Пуш в воскресенье/понедельник | Хук повторного вовлечения |
| Проекция даты | «При текущем темпе достигнешь цели к 15 мая» | Создаёт инвестицию в результат |

### Стратегия уведомлений (через Telegram-бот)

- **Напоминания о приёмах пищи**: «Не забудь внести обед! 🍱» в заданное время
- **Риск потери серии**: «Ты не вносил сегодня — не прерывай серию 14 дней!»
- **Еженедельное взвешивание**: Напоминание утром в выходной
- **Мотивация за достижение**: «Ты похудел на 2 кг! 🎉»
- **Реактивация**: После 3 дней без записей: «Мы скучаем! Твоя цель ждёт тебя»

### Ключевые выводы по UX

1. **Дневник — это и есть продукт.** Всё остальное его поддерживает. Должен открываться быстро и выглядеть наглядно.
2. **Кольцо калорий — важнейший визуальный элемент.** «Как я сегодня?» — за один взгляд.
3. **Снижение трения при логировании — главный приоритет.** Недавние блюда, фото, голос — чтобы добавить запись за 10 секунд.
4. **Серии — мощнейший механизм удержания** в категории (сильнее уведомлений и бейджей).
5. **График веса — эмоциональное ядро.** Нисходящая кривая — самое мотивирующее, что может показать приложение для похудения.
6. **Фото-ввод — уже стандарт** (MyFitnessPal, Lose It!, Yazio имеют его). Должен быть в 1 тап, не спрятан.

## 9. Технические ограничения Telegram Mini App

| Ограничение | Описание | Наше решение |
|---|---|---|
| Нет нативного сканера штрихкодов | `scanQrPopup` — только QR, не EAN | Пользователь отправляет фото упаковки → backend OCR |
| Камера только через file-upload | Нельзя управлять камерой программно | Стандартный Telegram file-picker с инструкцией |
| Шагомер iOS заблокирован | `DeviceMotionEvent` требует разрешения, в WebView не работает | Google Fit / Apple Health deep-link + ручной fallback |
| Нет push-уведомлений из Mini App | WebApp не может посылать уведомления | Бот-сообщения в заданное время |
| Хранение данных | Нет localStorage между сессиями | Backend + привязка к Telegram user_id |

### Что подтверждает исследование

1. **Ниша свободна** — нет ни одного качественного русскоязычного Telegram Mini App с фото-распознаванием + тренировками + шагомером
2. **Русская кухня** — незакрытый рынок для всех западных конкурентов (FatSecret, Calorie Mama, Lose It)
3. **Ближайший конкурент — Yoo.Health**, но у него нет тренировочного дневника, рецептов и нестабильное распознавание русской кухни
4. **Стек рабочий** — Claude Vision + python-telegram-bot + Railway закрывают MVP
5. **Монетизация реалистична** — $1,000–2,000/мес MRR достижимы при 2000–3000 активных пользователях
6. **Серии (streaks)** — топ-1 механик удержания в категории, обязательны с первого дня
7. **Точность порций** — главная техническая боль у всех конкурентов; нужно явно маркировать как «приблизительно» и давать легкий способ скорректировать

### Что нужно добавить в спецификацию

1. Сценарий ошибки распознавания (что делает бот, если блюдо не опознано)
2. Учёт часового пояса для вечерних отчётов
3. Сценарий возвращения пользователя после долгого отсутствия
4. Freemium-деление функций (что бесплатно, что платно)
5. UX для шагомера: ручной fallback + deep-link в Google/Apple Health
6. Экран «дефицит/профицит» — ключевой дифференциатор, которого нет ни у одного конкурента в Telegram
7. Счётчик серий с первого дня — без него retention упадёт критически
