# CalBot — План разработки

> Метод: вайбкодинг — код пишет Claude, человек проверяет результат в Telegram после каждого шага.
> Каждый этап даёт видимый результат. Следующий этап начинается только после проверки предыдущего.

## Этап 4 — Интуитивное питание

### Эмоциональный дневник питания
- [ ] При добавлении еды — опциональный вопрос "Что ты сейчас чувствуешь?"
- [ ] Набор эмоций кнопками: тревога / скука / радость / усталость / стресс / голод / привычка
- [ ] Сохранение эмоции вместе с записью о еде
- [ ] Еженедельный отчёт: какие эмоции чаще всего предшествуют еде
- [ ] AI-коуч анализирует паттерны и предлагает проработку ("Ты часто ешь при стрессе — вот что помогает")

### Режим без калорий

- [ ] Дневник показывает приёмы пищи и эмоции, без цифр

## UX-вопросы для проработки

- [ ] **Экран «Прогресс» в Mini App** — отдельный экран с графиком динамики: калории по дням (столбчатый или линейный), вес в динамике, процент дней в норме за неделю/месяц. Данные подтягиваются из истории дневника и весовых записей.

- [ ] **Составление меню в Mini App** — функция "Что приготовить?" и меню на день/неделю сейчас есть только в боте. Перенести в приложение: отдельный экран с кнопками "Меню на день" и "Меню на неделю", результат показывается прямо в приложении, можно добавить блюда в дневник одной кнопкой.

- [ ] **Расширение экрана «Добавить еду» в Mini App:**
  - **Фото из приложения** — кнопка "Сфотографировать", загрузка фото прямо из Mini App, передача в Claude Vision, автозаполнение КБЖУ
  - **Голосовой ввод** — кнопка микрофона, распознавание речи через Web Speech API, текст подставляется в поле «Что съела?»
  - **«Часто на завтрак»** и **«Часто на ужин»** — дополнительные табы в блоке «Повторить», показывают блюда отфильтрованные по приёму пищи

- [ ] **Умный ввод еды в Mini App** — сейчас ручное добавление требует ввести название и калории вручную. Нужно сделать умнее: пользователь пишет что съела (например "тарелка гречки с курицей"), Claude автоматически распознаёт калории и КБЖУ, предлагает заполненную карточку — пользователь просто подтверждает. Аналог фото-распознавания но через текст.

- [ ] **Кнопка меню рядом с кнопкой дневника в боте** — на главном экране бота сейчас есть кнопка перехода в Mini App (дневник). Хочется добавить рядом кнопку меню, но нужно подумать: что именно туда вынести, как не перегрузить экран, и как это соотносится с инлайн-кнопками которые уже есть.

- [ ] **Анимация обновления на главном экране** — после любого действия (еда, вода, шаги, тренировка) цифры баланса и потраченных калорий должны плавно обновляться: небольшая анимация кольца и счётчиков вместо мгновенной замены значений.

- [ ] **Функция «Своя тренировка» в боте** — сейчас реализована как отдельный тип наравне с Кардио/Силовой/Йогой/Растяжкой, но непонятно как пользователь должен её использовать. Нужно переосмыслить: возможно, это должно быть поле ввода названия тренировки + калории вручную, или вопрос «сколько минут занималась?» без выбора типа.

- [ ] **Добавление шагов в Mini App** — сейчас реализованы два варианта, нужно выбрать лучший:
  - **Вариант A (текущий):** сетка быстрых кнопок (1000 / 3000 / 5000 / 8000 / 10000 / 12000 / 15000 / ✏️ своё) — тап сразу сохраняет
  - **Вариант B (старый):** поле ввода числа + кнопка "Сохранить"
  - Вопрос: быстрые варианты удобны когда шаги круглые, но реальное число шагов всегда произвольное (7348, 9201...). Возможно, лучше показывать число из шагомера телефона точно.
  - Рассмотреть: можно ли подтянуть шаги автоматически из Health/Google Fit
