# CalBot — План разработки

> Метод: вайбкодинг — код пишет Claude, человек проверяет результат в Telegram после каждого шага.
> Каждый этап даёт видимый результат. Следующий этап начинается только после проверки предыдущего.

## UX-вопросы для проработки

- [x] **Экран «Прогресс» в Mini App** — отдельный экран с графиком динамики: калории по дням (столбчатый или линейный), вес в динамике, процент дней в норме за неделю/месяц. Данные подтягиваются из истории дневника и весовых записей.

- [x] **Составление меню в Mini App** — функция "Что приготовить?" и меню на день/неделю сейчас есть только в боте. Перенести в приложение: отдельный экран с кнопками "Меню на день" и "Меню на неделю", результат показывается прямо в приложении, можно добавить блюда в дневник одной кнопкой.

- [x] **Расширение экрана «Добавить еду» в Mini App:**
  - [x] **Фото из приложения** — кнопка 📸, ресайз в браузере, передача в Claude Vision, AI-превью с КБЖУ
  - [x] **Голосовой ввод** — кнопка 🎤, Web Speech API (ru-RU), текст подставляется в поле описания
  - [ ] **«Часто на завтрак»** и **«Часто на ужин»** — дополнительные табы в блоке «Повторить», показывают блюда отфильтрованные по приёму пищи

- [x] **Уведомления через кнопку в боте и в приложении** — кнопка «🔔 Уведомления» добавлена в главное меню бота; в Mini App — отдельный экран через «Добавить → Напоминания» с просмотром, добавлением и удалением; REST API `/api/notifications`.

- [x] **Кнопка перехода в Mini App переименована** — «Дневник» → «📱 В приложение»; устанавливается через `set_chat_menu_button` при запуске бота.

- [x] **«Меню на день/неделю» переименовано в «Рацион»** — убрана путаница между навигационным меню и планом питания.

- [x] **Кнопка «Статистика» добавлена в главное меню** — рядом с «Уведомлениями».

- [x] **Анимация обновления на главном экране** — после любого действия (еда, вода, шаги, тренировка) цифры баланса и потраченных калорий должны плавно обновляться: небольшая анимация кольца и счётчиков вместо мгновенной замены значений.

- [x] **Функция «Своя тренировка» в боте** — реализована: кнопка «⚡️ Свой вид спорта» открывает текстовое поле, пользователь вводит название (теннис, бокс, танцы и т.д.), далее выбирает длительность как обычно. Своё название сохраняется в БД, калории считаются по 5 ккал/мин.

- [x] **Запись веса прямо из экрана Прогресс** — сейчас вес добавляется только через команду /weight в боте. Добавить кнопку "Записать вес" прямо на экране Прогресс: поле ввода числа + сохранение через API.

- [x] **Добавление еды через фото, голос и штрихкод в Mini App** — реализовано:
  - [x] **Фото** — 📸 Фото: ресайз до 1280px в браузере, base64 → /api/food/photo, AI-превью КБЖУ, одна кнопка подтверждает
  - [x] **Голос** — 🎤 Голос: Web Speech API (ru-RU), распознанный текст → поле описания → /api/food/estimate
  - [x] **Штрихкод** — 📦 Штрихкод: фото упаковки → Claude находит штрихкод → Open Food Facts (данные на 100г) → поле «Сколько грамм?» → пересчёт калорий

- [x] **Подсказки о способах добавления еды в боте** — сейчас бот пишет "Сфотографируй блюдо или опиши текстом". Нужно кратко упомянуть все доступные способы: фото блюда, фото штрихкода, голосовое сообщение, текст. Обновить текст подсказки в обоих местах: кнопка "Добавить еду" и сценарий ошибки распознавания.

- [x] **Настройка уведомлений** — пользователь выбирает, когда получать напоминания: утреннее (записать завтрак), вечернее (итоги дня), питьевой режим. Время и набор уведомлений настраивается через профиль или отдельную команду `/notifications`.
