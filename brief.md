# CalBot Mini App — Brief v1

> Цель: визуальный дневник питания внутри Telegram.
> Добавление еды остаётся в боте — Mini App только показывает и анализирует.
> Стек: React + Vite + @telegram-apps/telegram-ui

---

## Философия дизайна

- Выглядит как нативное приложение Telegram, не как сайт
- Минимум текста — максимум визуала (кольца, плитки, цвет)
- Все кнопки минимум 44px в высоту
- Русский язык интерфейса
- Поддержка светлой и тёмной темы Telegram автоматически

## Элементы на экране

### Шапка
- Текущий день недели + дата
- Без навигации (только сегодня в v1)

### Кольцо прогресса (Progress Ring)
- Круговой прогресс-бар: заполнение = съедено / норма
- Цвет: зелёный (в норме) → оранжевый (80–100%) → красный (перебор)
- В центре: съеденные ккал крупно, норма мелко под ним
- Под кольцом: "Осталось N ккал" или "Перебор на N ккал"

### Плитки КБЖУ
- 3 плитки в ряд: Белки / Жиры / Углеводы
- Каждая: иконка + граммы + название
- Компактно, без лишних деталей

### Блоки приёмов пищи
- 4 блока: 🌅 Завтрак / ☀️ Обед / 🌙 Ужин / 🍎 Перекус
- Каждый блок: заголовок + сумма ккал справа
- Под заголовком: список блюд (название, без КБЖУ)
- Если приём пуст: серый текст "Ещё не добавлено"
- Блоки всегда показываются все 4, даже пустые

### Кнопка внизу (фиксированная)
- "＋ Добавить через бот" — открывает чат с ботом
- Всегда видна, прилипает к низу экрана
- Крупная, на всю ширину

## Действия пользователя

| Действие | Результат |
|---|---|
| Открыть Mini App | Загружается дневник за сегодня |
| Смотреть кольцо | Визуально понять прогресс |
| Скроллить вниз | Просматривать приёмы пищи |
| Нажать "Добавить через бот" | Открывается чат с ботом |

## Технические требования

### Frontend
- React 18 + Vite + TypeScript
- @telegram-apps/sdk-react — получение user_id из Telegram
- @telegram-apps/telegram-ui — нативные компоненты Telegram
- Без дополнительных CSS-фреймворков (только Telegram UI)

### Backend (добавляется к боту)
- Новый эндпоинт: `GET /api/diary?user_id=XXX&date=YYYY-MM-DD`
- Возвращает: записи за день, норму калорий пользователя
- Аутентификация: проверка Telegram initData

### Деплой
- Frontend: Vercel (бесплатно, автодеплой из GitHub)
- Backend: тот же Render (добавляем эндпоинты к боту)

## Порядок разработки

1. Добавить API-эндпоинт `/api/diary` к боту
2. Настроить React-проект (уже создан в `miniapp/`)
3. Сделать компонент кольца прогресса
4. Сделать плитки КБЖУ
5. Сделать блоки приёмов пищи
6. Подключить реальные данные из API
7. Задеплоить на Vercel
8. Подключить в BotFather как Mini App
